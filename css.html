Historien om CSS

Den første udgave af CSS blev udviklet af Håkon Wium Lie som arbejde i CERN, men den blev overtaget af w3c som overtog standardiseringsarbejdet. Den første reele browser der understøttede det var IE3, deefter overtog netscape 4,0 men der var en masse fejl.

CSS er kun blevet bedre siden den første version og den understøttes nu af alle store browseres standarder.

CSS i HTML

Når jeg vil bruge CSS til at designe min HTML-fil kan jeg vælge at have en intern stylesheet eller et i et eksternt dokument.

For at kunne designe min HTML skal jeg bruge de elementer/tags jeg har skrevet i HTML som f.eks. body, h1 og sektion. Hvis jeg vil ændre flere elementer på én gang kan jeg bruge bestemte selectors som f.eks. class og hvis jeg kun vil ændre ét element kan jeg bruge id-selectoren.

For at kunne bruge min CSS skal jeg kode på en bestemt måde som består af en række deklerations (se billede).

De forskellige selectors

Der er mange forskellige selectors at vælge imellem, men jeg har valgt kun at nævne de tre vigstigste selcetors her. Hvis jeg vil vide mere, kan jeg tjekke punktet: "HTML og CSS-workshop".

De tre vigtigste selectors er:

  • Element selector
  • id selector
  • class-selector

    • Hvis jeg vil skrive en element selctor i CSS skal jeg f.eks. skrive h1{design}.

    Hvis jeg skal skrive en id-selctor i css skal jeg altid skrive # foran navnet på den bestemte id f.eks.: #id eller #andetnavn{design}

    Hvis jeg skal skrive en class skal jeg have punktom foran navnet f.eks. .class{design} eller .navn{design}.

    Der findes også nogle pseudo-class selectors som f.eks. "first child". (HTML-og CSS workshop)

    Kaskaderegler

    I CSS finders der nogle kaskaderegler der bestemmer hvilken kodning der gælder.

    Hvis jeg f.eks. skriver, at h1 skal baggrundsfarven blå, men senere hen skriver at baggrunden skal være grøn, er det den sidste kodning der gælder, det hedder "source order"

    Hvis jeg skal skrive meget specifikt hvordan h1 ser ud, er den specifikke kodning der gælder, det kaldes "specificity". Den overwriter også source order.

    Kasser i HTML

    Hvis jeg vil have kasser i HTML, der repræsentere HTML-elemter findes to typer:

  • inline, som gør at HTML-elementerne placeres horisontalt efter hinanden.
  • block, som placere elementerne efter hinanden i stakker.

    • De styres med CSS-properties, display og block samt inline

    Du kan også vælge forskellige egenskaber til dine kasser.

    Kasserne kan laves med padding, som er afstand imellem indholdet og rammen. Ramme/border, som kan være synlige eller usynlige og margen, som er afstand imellem rammen og de andre elementer.

    Farver

    Jeg kan ændre farver på mine HTML-elementer ved at skrive elementets navn og en farvekode.

    Der findes flere forskellige farvekoder men dem jeg bruger er er enten RGB som skrives rgb(255,204,0) og hex kodensom skrives #ffcc00.

    Jeg kan også ændre på elementets gennemsigtighed med opacity.

    Position af HTML-elementer

    Hvis jeg vil have nogle bestemte HTML-dokumenter placeret bestemte steder, kan jeg bruge forskellige positioneringer.

    Der er f.eks. 'flow", der gør at elementet flyder fra toppen og ned, og "relative", som er et forskudt flow.

    Hvis jeg vælger "absolute" vil elementerne ikke flyde på siden, men følge med når jeg scroller op og ned.

    For at fastgøre et element bruges "fixed", som gør at elementet ikke flytter med ved scroll og sidst men ike mindst er der "float", som gør, at elementet "flyder" i det overordnede element og hvilken side elemntet skal flyde i. Den bruges ofte til at placere billeder i tekst og til ar placere elementer ved siden af hinanden

    Hvis jeg vil ophæve float, kan jeg bruge "clear" som kan være både højre, venstre, begger eller ingen, og kan afgrænse hvilke sider der må røre andre elementer.
